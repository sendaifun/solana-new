# Phase Handoff Specification

Defines the structured markdown context files that skills write to `.superstack/` in the project workspace. Each phase's output is the next phase's input.

## File Locations

```
<project-root>/
  .superstack/
    idea-context.md       # Written by Phase 1 (Idea) skills
    build-context.md      # Written by Phase 2 (Build) skills
    learnings.md          # Written by /learn skill
```

- Skills that run later should merge into the existing file, not overwrite it.

## Creating Context Files

Any skill can create a context file if it doesn't exist yet. The user may invoke skills in any order — they don't need to follow the Learn → Idea → Build → Launch sequence.

When a skill needs a context file that doesn't exist:

1. **Proceed immediately.** Ask the user directly for the information you need.
2. **Create the file** with whatever context you gather through conversation. Use the format above. Fill in what you know, leave other sections out.
3. Do NOT redirect the user to run other commands first.
4. Do NOT warn about missing files or print dependency chains.

This way, whichever skill runs first bootstraps the context file for all subsequent skills.

## Merging Rules

When a skill updates an existing context file:

1. Read the current file first.
2. Add new sections or update existing ones. Do not remove sections written by other skills.
3. List fields (Skills Installed, Milestones, Findings, Source Reports) should be appended to, not replaced.
4. Scalar fields (scores, statuses) should be overwritten with the latest value.
5. Always update `Completed at` with the current timestamp.
