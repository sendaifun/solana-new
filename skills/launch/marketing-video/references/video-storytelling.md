# Video Storytelling Frameworks

Six video narrative structures optimized for crypto/Solana marketing. Each includes a script template you can fill in.

---

## 1. Hook-Proof-CTA (Social Media Standard)

**Best for:** Twitter, TikTok, Instagram (15-30s)
**Why it works:** Social algorithms reward retention. The hook stops the scroll, the proof holds attention, the CTA converts.

| Beat | Duration | Content |
|------|----------|---------|
| **Hook** | 1-3s | Shocking metric, bold claim, or visual interrupt. Must work WITHOUT sound. |
| **Proof** | 10-20s | Product demo, screen recording, or data visualization backing up the hook. |
| **CTA** | 3-5s | One clear action: "Try it now", "Link in bio", QR code. |

### Script Template
```
HOOK (1-3s):
"[METRIC] in [TIMEFRAME]."
Example: "$2.1M settled in 30 days. Zero bank accounts."

PROOF (10-20s):
[Screen recording of the product doing the thing]
[Text overlay explaining what's happening]
[Key metric callout at the peak moment]

CTA (3-5s):
"Try it free → [URL]"
[QR code to product]
[Logo + tagline]
```

### Remotion Structure
```
Scene 1 (0-90 frames):   Bold text animation with metric
Scene 2 (90-600 frames):  Product screenshots with animated annotations
Scene 3 (600-750 frames): Metric callout with spring animation
Scene 4 (750-900 frames): CTA with QR code + URL
```

## 2. Problem-Demo-Result (Product Demo Standard)

**Best for:** Product demos, landing pages (30-90s)
**Why it works:** Creates emotional contrast. Viewer feels the pain, sees the solution, wants the result.

| Beat | Duration | Content |
|------|----------|---------|
| **Problem** | 5-10s | Show the current painful way of doing things. Frustration, wasted time, lost money. |
| **Demo** | 15-45s | Your product solving the problem. Step by step. Real UI, real data. |
| **Result** | 5-10s | The outcome: time saved, money earned, friction eliminated. One clear metric. |

### Script Template
```
PROBLEM (5-10s):
"Right now, [TARGET USER] has to [PAINFUL PROCESS]."
"That costs [TIME/MONEY] and fails [FREQUENCY]."
[Show: screenshots of the old/bad way]

DEMO (15-45s):
"With [PRODUCT], you just [SIMPLE ACTION]."
[Screen recording: step 1 → step 2 → step 3]
[Text overlays: "One click", "400ms", "$0.002 fee"]
[Highlight the "aha" moment with a zoom or annotation]

RESULT (5-10s):
"[USER] saves [METRIC] every [PERIOD]."
[Before/after split screen comparison]
[CTA: "Start free → URL"]
```

## 3. Before/After (Transformation Story)

**Best for:** Landing pages, comparison content (30-60s)
**Why it works:** Contrast is the most powerful visual persuasion tool. Side-by-side or sequential transformation.

| Beat | Duration | Content |
|------|----------|---------|
| **Before** | 10-20s | The old way. Slow, expensive, frustrating. Use real examples. |
| **Transition** | 2-3s | Visual wipe, split screen, or "introducing..." moment. |
| **After** | 10-20s | Your product. Fast, cheap, delightful. Same task, different experience. |
| **CTA** | 5s | Clear action. |

### Visual Techniques
- **Split screen:** Left = old way (red-tinted, slow motion), Right = your way (green-tinted, full speed)
- **Sequential:** Show old way first, then hard cut to your way
- **Overlay counter:** Timer running on both sides showing time difference
- **Cost ticker:** Running cost comparison (old way: $4.50... your way: $0.002)

### Script Template
```
BEFORE (10-20s):
[Title: "The old way"]
[Screen recording of competitor/manual process]
[Timer overlay: counting up slowly]
[Cost overlay: "$4.50 in fees..."]
[Red-tinted, slightly desaturated color grade]

TRANSITION (2-3s):
[Wipe transition or "There's a better way" text]

AFTER (10-20s):
[Title: "[PRODUCT NAME]"]
[Screen recording of your product doing the same thing]
[Timer overlay: "0.4 seconds"]
[Cost overlay: "$0.002"]
[Full color, vibrant, Solana purple/green accents]

CTA (5s):
"Switch now → [URL]"
```

## 4. Speedrun (Hackathon Demo Format)

**Best for:** Hackathon demos, developer content (1-3min)
**Why it works:** Developers respect speed and capability. Showing the full flow proves completeness.

| Beat | Duration | Content |
|------|----------|---------|
| **Setup** | 10-15s | What you're about to build/show and why it matters |
| **Speedrun** | 60-120s | Full product walkthrough at 1.5-2x speed with narration |
| **Architecture** | 15-30s | Quick tech stack / how it works under the hood |
| **Metrics** | 10-15s | Key numbers: users, transactions, performance benchmarks |
| **CTA** | 10s | GitHub link, live demo link, "Built at [Hackathon]" |

### Script Template
```
SETUP (10-15s):
"In [HACKATHON_TIME], we built [PRODUCT] — [ONE SENTENCE DESCRIPTION]."
"Let me show you how it works."

SPEEDRUN (60-120s):
[Screen recording at 1.5x speed]
[Narration or text overlays at each step]
Step 1: "Connect wallet" [2s]
Step 2: "Select action" [3s]
Step 3: "Confirm transaction" [2s]
Step 4: "Done — 400ms" [2s]
[Show terminal output, explorer links, real transaction hashes]

ARCHITECTURE (15-30s):
[Simple diagram or code snippet]
"Built with [STACK]: Anchor + React + [KEY_TECH]"
"[KEY_TECHNICAL_ACHIEVEMENT] — here's how..."

METRICS (10-15s):
"Since deploying: [METRIC_1], [METRIC_2], [METRIC_3]"
[Animated counters]

CTA (10s):
[GitHub logo] "github.com/[REPO]"
[Product URL]
"Built at [HACKATHON] by [TEAM]"
```

## 5. Testimonial Sandwich (Social Proof Format)

**Best for:** YouTube, longer content, fundraising videos (60-180s)
**Why it works:** Third-party validation is more persuasive than self-promotion.

| Beat | Duration | Content |
|------|----------|---------|
| **User quote** | 10-15s | Real user describing the problem or their experience |
| **Product demo** | 30-90s | Your product solving what the user described |
| **User quote** | 10-15s | Same or different user describing the result |
| **CTA** | 5-10s | Clear action |

### Tips
- Use real quotes from users (Discord, Twitter, user interviews)
- Text overlay quotes work if you don't have video testimonials
- Show the user's Twitter handle or Discord username for authenticity
- Multiple short quotes (3-5s each) can replace one long testimonial

## 6. Data Story (Metrics-Driven Narrative)

**Best for:** DeFi products, infrastructure, investor-facing content (30-90s)
**Why it works:** Numbers are the most compelling content for crypto-native audiences. Animated data is mesmerizing.

| Beat | Duration | Content |
|------|----------|---------|
| **The number** | 3-5s | One striking metric, animated large |
| **The context** | 10-15s | What that number means, visualized as a chart or comparison |
| **The growth** | 10-20s | Animated chart showing trajectory over time |
| **The engine** | 10-15s | Brief explanation of what drives the growth (your product) |
| **CTA** | 5s | Clear action |

### Remotion Techniques for Data Stories
- **Animated bar charts** with staggered spring entrance (each bar springs up sequentially)
- **Line charts** using `@remotion/paths` with `evolvePath` for progressive drawing
- **Counter animations** using `interpolate()` on number values
- **Pie chart reveals** using `stroke-dashoffset` animation
- **Comparison metrics** with animated scale/balance visuals

### Script Template
```
THE NUMBER (3-5s):
[Large animated text: "$47M"]
[Subtitle fades in: "settled on-chain this month"]

THE CONTEXT (10-15s):
[Bar chart comparing: Your protocol vs competitors]
[Or: pie chart showing market share]
[Text: "That's 3x more than last month"]

THE GROWTH (10-20s):
[Line chart animating from month 1 to now]
[Key inflection points annotated]
[Growth rate callout: "42% MoM"]

THE ENGINE (10-15s):
[Brief product screenshot or diagram]
[Text: "Powered by [PRODUCT] — [ONE LINER]"]

CTA (5s):
[Logo + URL + QR code]
```

## Choosing the Right Framework

```
Is your audience developers or crypto-native?
  → Social media post? → Hook-Proof-CTA
  → Hackathon demo? → Speedrun
  → DeFi/infra product? → Data Story

Is your audience general/consumer?
  → Have a demo ready? → Problem-Demo-Result or Before/After
  → Have user testimonials? → Testimonial Sandwich
  → Landing page hero? → Before/After

Is this for investors?
  → Data Story for metrics-heavy products
  → Testimonial Sandwich for user-love stories
  → Problem-Demo-Result for clear product demos
```

## The 3-Second Rule

Regardless of framework, the first 3 seconds determine everything:
- **Twitter:** 1.5s before users scroll past
- **TikTok:** 1s before swipe
- **YouTube:** 5s before click away
- **Landing page:** 3s before bounce

Your hook must work WITHOUT sound. 85% of social video is watched muted. Use:
- Large bold text with the key metric
- A surprising visual (speed of transaction, split-screen contrast)
- A pattern interrupt (unexpected color, motion, or format)
