# Professional Video Quality Guide — Don't Look AI-ish

How to make Remotion videos that look hand-crafted by a motion designer, not generated by a prompt. These patterns separate amateur/AI output from professional video.

---

## The 15 Tells of AI-Generated Video

These are the patterns that instantly make a video feel "AI-made." Avoid all of them:

1. **Linear interpolation everywhere** — No easing, robotic movement. Every value change needs easing.
2. **Kitchen-sink animation** — Too many different animation styles crammed together. Pick 2-3 techniques and use them consistently.
3. **Default bounce on everything** — Bouncy text, bouncy backgrounds, bouncy everything. Bounce is for intentional playfulness, not a default.
4. **No staggering** — Everything appears at once instead of choreographed entrance. Elements need hierarchy.
5. **No dwell time** — Elements animate in and immediately animate out. Content needs breathing room.
6. **Per-character opacity for typewriter** — Use string slicing instead. Per-character opacity looks synthetic.
7. **CSS animations** — Causes flickering in Remotion renders. All motion MUST be frame-driven.
8. **Text outside safe zones** — Content cut off by platform UI (status bars, navigation).
9. **Tiny fonts** — Anything under 28px is unreadable on mobile. Headlines need 56px+.
10. **No audio at all** — Silent videos feel sterile, even for muted-first platforms (add SFX).
11. **Static elements after entrance** — No subtle continuous motion. Resting elements look dead.
12. **Hard cuts between scenes** — No transitions, light leaks, or overlays at cut points.
13. **Unclamped interpolation** — Values overshoot past their intended range, causing subtle glitches.
14. **Same timing for every element** — Professional videos vary timing by element importance.
15. **Flat unchanging colors** — No dynamic color shifts, no depth, no visual progression.

## Disney's 12 Principles (Applied to Remotion)

These principles were designed for character animation but apply directly to motion graphics:

### 1. Ease In / Ease Out (Most Important)
Nothing in the real world starts or stops instantly. Every motion needs acceleration and deceleration.

```tsx
// BAD — linear, robotic
const x = interpolate(frame, [0, 30], [0, 100]);

// GOOD — eased, natural
const x = interpolate(frame, [0, 30], [0, 100], {
  easing: Easing.inOut(Easing.cubic),
  extrapolateRight: "clamp",
});

// BEST — spring physics, organic
const x = spring({ frame, fps, config: { damping: 200 } }) * 100;
```

### 2. Staging
Direct the viewer's attention. One focal point at a time. Don't animate 5 things simultaneously — animate the most important element first, then reveal secondary elements.

```tsx
// Stagger reveals by importance
<Sequence from={0}><Headline /></Sequence>      {/* Most important first */}
<Sequence from={20}><Subtitle /></Sequence>      {/* Secondary, 0.67s later */}
<Sequence from={45}><MetricCards /></Sequence>    {/* Supporting, 1.5s later */}
<Sequence from={70}><CTAButton /></Sequence>      {/* Action, 2.3s later */}
```

### 3. Overlapping Action
Related elements move at different rates. The image arrives first, then the title follows, then the description trails behind. This creates visual depth and realism.

```tsx
const imageScale = spring({ frame, fps, config: { damping: 200 } });
const titleOpacity = spring({ frame: frame - 8, fps, config: { damping: 200 } });
const descOpacity = spring({ frame: frame - 16, fps, config: { damping: 200 } });
```

### 4. Follow-Through
When an object stops moving, secondary elements continue briefly. A card lands, then its shadow settles. Text arrives, then its underline slides in.

### 5. Arcs
Real objects follow curved paths, not straight lines. Use combined translateX + translateY with different timing for arc motion:

```tsx
const progress = spring({ frame, fps, config: { damping: 200 } });
const x = interpolate(progress, [0, 1], [-200, 0]);
const y = interpolate(progress, [0, 0.5, 1], [0, -60, 0]); // Arc upward
```

### 6. Secondary Action
Background elements have subtle motion that supports the main action without competing for attention. A gradient slowly shifts, particles drift, a glow pulses.

## The Consistent Animation Vocabulary

The #1 difference between professional and amateur motion graphics: **consistency**.

### Pick Your Palette
Choose 2-3 animation techniques and use them throughout the entire video:

**Minimal palette (recommended):**
- Entrance: Fade up + slide from bottom (smooth spring, damping 200)
- Emphasis: Scale pulse (snappy spring, damping 20)
- Exit: Fade out

**Energetic palette:**
- Entrance: Spring scale from 0 (bouncy, damping 8)
- Emphasis: Color shift with `interpolateColors()`
- Exit: Slide out + fade

**Cinematic palette:**
- Entrance: Slow fade + slight zoom (heavy spring, damping 15, mass 2)
- Emphasis: Light leak overlay
- Exit: Dissolve (fade with blur increase)

### Anti-Pattern: The Animation Variety Show
```tsx
// BAD — every element uses a different animation style
<Scene1> {/* Spring bounce entrance */}
<Scene2> {/* Typewriter text */}
<Scene3> {/* 3D flip */}
<Scene4> {/* Wipe reveal */}
<Scene5> {/* Elastic bounce */}
```

```tsx
// GOOD — consistent vocabulary throughout
<Scene1> {/* Fade up + slide, spring damping 200 */}
<Scene2> {/* Fade up + slide, spring damping 200, staggered */}
<Scene3> {/* Fade up + slide, spring damping 200, staggered */}
<Scene4> {/* Fade up + slide, spring damping 200, with metric scale emphasis */}
<Scene5> {/* Fade up + slide, spring damping 200 */}
```

## Spring Physics — The Professional Default

### The Golden Rule: No Bounce by Default
`damping: 200` should be your default for everything. Bounce (`damping: 8`) is a deliberate creative choice, not a default. Most professional motion graphics have zero visible wobble.

### When to Use Each Preset

| Preset | Config | Use For | Feel |
|--------|--------|---------|------|
| **Smooth** | `{ damping: 200 }` | 90% of all animations | Elegant, professional |
| **Snappy** | `{ damping: 20, stiffness: 200 }` | Quick reveals, buttons | Responsive, crisp |
| **Bouncy** | `{ damping: 8 }` | Logos, playful emphasis | Fun, attention-grabbing |
| **Heavy** | `{ damping: 15, stiffness: 80, mass: 2 }` | Dramatic reveals, hero elements | Weighty, cinematic |

### Tuning Order
1. **Damping first** — Controls how quickly motion settles. Higher = less bounce.
2. **Stiffness second** — Controls responsiveness. Higher = snappier start.
3. **Mass last** — Controls weight. Higher = slower, more deliberate.

## Timing and Rhythm

### Multi-Point Keyframes (Dwell Time)
Elements need time to be read. The pattern is: enter → hold → exit.

```tsx
// Element fades in, holds for 2 seconds, then fades out
const opacity = interpolate(
  frame,
  [0, 20, 80, 100],  // in over 20f, hold 60f, out over 20f
  [0, 1, 1, 0],
  { extrapolateRight: "clamp" }
);
```

### Time in Seconds, Not Frames
Write animations relative to fps for portability:

```tsx
const { fps } = useVideoConfig();
// "Fade in over 0.5 seconds" — works at any frame rate
const opacity = interpolate(frame, [0, 0.5 * fps], [0, 1], {
  extrapolateRight: "clamp",
});
```

### Stagger Spacing Guide
- **Tight stagger (5-8 frames):** Related items in a list, metric cards
- **Medium stagger (10-15 frames):** Section elements, headline then body
- **Wide stagger (20-30 frames):** Scene changes, major reveals

## Typography Rules for Video

### Minimum Sizes
- **Headlines:** 56px minimum (72-144px recommended)
- **Body text:** 36px minimum
- **Captions/labels:** 28px absolute minimum
- **Code/terminal:** 28px minimum with monospace font

### Mobile Safe Zones
All text and key content must stay within safe zones on vertical (9:16) formats:
- **Top:** At least 150px from top edge (platform search bars, status bar)
- **Bottom:** At least 170px from bottom edge (navigation, swipe-up UI)
- **Sides:** At least 40px padding

### Font Pairing
- **Headlines:** Inter (weight 700-900) or custom brand font
- **Body:** Inter (weight 400-500)
- **Code/terminal:** JetBrains Mono
- Never use more than 2 font families in a single video

### Text Readability
- White text on dark backgrounds (not gray-on-gray)
- Add `textShadow: "2px 2px 8px rgba(0,0,0,0.6)"` for text over images/video
- Limit to 6-8 words per line for readability at speed
- One idea per text card — don't stack paragraphs

## Color Theory for Video

### Dynamic Color (Not Flat)
Use `interpolateColors()` to shift colors over time. Even subtle shifts add life:

```tsx
import { interpolateColors } from "remotion";

// Background shifts from deep purple to dark blue across the scene
const bg = interpolateColors(
  frame,
  [0, durationInFrames],
  ["#1a0533", "#0a1628"]
);
```

### Depth Through Gradients
Layer multiple gradients for dimensional backgrounds:

```tsx
background: `
  radial-gradient(ellipse at 30% 40%, rgba(153, 69, 255, 0.15) 0%, transparent 60%),
  radial-gradient(ellipse at 70% 60%, rgba(20, 241, 149, 0.08) 0%, transparent 50%),
  linear-gradient(180deg, #0a0a0f 0%, #12101a 100%)
`
```

### Color Grading for Mood
- **Before scenes:** Desaturated, slightly red-tinted, low contrast
- **After scenes:** Full saturation, Solana purple/green accents, high contrast
- **Metric reveals:** Bright accent color (#14F195) against dark background
- **CTAs:** High contrast, brand colors at full saturation

## Organic Motion (The Anti-Robot Pattern)

### Continuous Subtle Motion
After entrance animations complete, elements should NOT be perfectly static. Add barely-perceptible drift:

```tsx
import { noise3D } from "@remotion/noise";

// Gentle floating motion for a resting element
const floatX = noise3D("x", 0, 0, frame * 0.005) * 3;  // +-3px drift
const floatY = noise3D("y", 0, 0, frame * 0.005) * 3;

<div style={{ transform: `translate(${floatX}px, ${floatY}px)` }}>
  {children}
</div>
```

### Breathing Backgrounds
Subtle pulsing glow or gradient shift that runs continuously:

```tsx
const breathe = Math.sin(frame * 0.02) * 0.5 + 0.5; // 0-1 oscillation
const glowOpacity = interpolate(breathe, [0, 1], [0.05, 0.12]);

<div style={{
  background: `radial-gradient(circle at center, rgba(153,69,255,${glowOpacity}) 0%, transparent 70%)`,
}} />
```

### Particle Drift
Background particles that move with noise create ambient visual texture:

```tsx
// Generate 20 particles with noise-driven positions
{Array.from({ length: 20 }).map((_, i) => {
  const x = noise3D("px", i, 0, frame * 0.003) * 0.5 + 0.5;
  const y = noise3D("py", 0, i, frame * 0.003) * 0.5 + 0.5;
  const opacity = noise3D("po", i, i, frame * 0.005) * 0.3 + 0.3;
  return (
    <div key={i} style={{
      position: "absolute",
      left: `${x * 100}%`,
      top: `${y * 100}%`,
      width: 4, height: 4, borderRadius: "50%",
      backgroundColor: "#9945FF", opacity,
    }} />
  );
})}
```

## The Professional Quality Checklist

Run through this before every render:

### Motion
- [ ] Default spring is `damping: 200` (no bounce) — bounce only where intentional
- [ ] All elements stagger entrance (10-20 frame gaps)
- [ ] Multi-point keyframes include dwell time (enter, hold, exit)
- [ ] Consistent animation vocabulary — max 2-3 techniques throughout
- [ ] All interpolations use `extrapolateRight: "clamp"`
- [ ] No CSS transitions or Tailwind animate classes
- [ ] Resting elements have subtle continuous motion (noise/sine)

### Visual
- [ ] Dynamic background (gradient shifts, breathing glow, particle drift)
- [ ] Color progression across scenes (not flat single color)
- [ ] Headlines 56px+, body 36px+, minimum 28px
- [ ] Mobile safe zones respected (150px top, 170px bottom for 9:16)
- [ ] Max 2 font families
- [ ] Text shadow on any text over images/video
- [ ] 6-8 words per line maximum

### Transitions
- [ ] TransitionSeries with springTiming between scenes (not hard cuts)
- [ ] Light leak overlays at key scene changes
- [ ] Whoosh SFX on transitions, ding on reveals
- [ ] Transition duration 15-30 frames (not too fast, not too slow)

### Polish
- [ ] Premount all Sequences with heavy content
- [ ] Audio (at minimum SFX on transitions and reveals)
- [ ] Render at appropriate quality (CRF 18 for final, bt709 color space)
- [ ] First 3 seconds have a compelling hook (works without sound)
- [ ] Last 3 seconds have a clear, single CTA
