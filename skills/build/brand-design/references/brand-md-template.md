## Usage dos and don'ts

**Do:**

- Use the shadcn CSS variables (`bg-background`, `text-foreground`, `bg-primary`) everywhere. Never hardcode hex.
- Keep the 4 px spacing grid (`gap-2`, `p-4`, `px-6`).
- Respect `prefers-color-scheme` by using the `.dark` class on `<html>` based on the system setting.
- Use `font-mono tabular-nums` for any number that updates in place (balances, prices, counters).
- Test every component in both light and dark mode before considering it done.

**Don't:**

- Hardcode colors (`#FF0000`, `rgb(...)`) in component files.
- Use `transition: all` — always specify `transition-colors`, `transition-transform`, etc.
- Mix more than one gradient on the same screen.
- Use the accent gradient as a background for body text.
- Override shadcn tokens per-component — if a color is wrong, fix it in `globals.css` once.

## How the rest of the toolchain uses this file

- `frontend-design-guidelines` reads this file at the top of its workflow and uses the palette + typography info when generating any new component.
- If you run `scaffold-project` again or re-theme, back up this file first (`cp brand.md brand.md.bak`).
- If you open a PR adding a new component, it should visually match this palette — reviewers can reference this file.

## Mood → voice mapping

The "Tone and voice" section is auto-generated based on the palette's mood. Use this table as the source.

### technical / serious (defi, infra, ai)

**Words to use:** direct, specific, number-forward, factual. Prefer verbs over adjectives. Prefer nouns over metaphors. When you describe an action, say what it does, not what it feels like.

**Words to avoid:** hype words (revolutionary, game-changing, unleash, supercharge), emojis, exclamation marks, overpromising. "Unlock" is banned.

**Voice example:** "Supplied 1,000 USDC at 4.2% APY. Withdraw anytime — no lockup."

### minimal (infra, ai, tooling)

**Words to use:** sparse, precise, active voice. One sentence per idea. No filler. Lists over paragraphs. Numbers over adjectives.

**Words to avoid:** "simply", "just", "easy", "powerful", "seamless", any word that describes the effort instead of the outcome.

**Voice example:** "42 ms. 200 MB. 0 errors."

### premium (nft, ai, high-end consumer)

**Words to use:** confident, understated, considered. Lean into whitespace and short sentences. Describe things with restraint — "considered" not "amazing", "crafted" not "built with love".

**Words to avoid:** "luxury", "exclusive", "elite" (using them makes it less premium, not more). Avoid stacking adjectives.

**Voice example:** "A place for work that deserves a second look."

### warm (consumer, social, lifestyle)

**Words to use:** human, friendly, encouraging. Second person ("you"). Contractions are fine. Questions are fine. Metaphors are fine as long as they're not cliché.

**Words to avoid:** "utilize" (say "use"), "leverage", "synergy", corporate verbs, exclamation marks outside of genuine moments.

**Voice example:** "Send your friend some sats. They'll thank you later."

### playful / bold (memecoin, community)

**Words to use:** loose, self-aware, internet-native. Lowercase is fine. Memes are fine in moderation. Specific jokes land better than generic enthusiasm.

**Words to avoid:** corporate tone, trying too hard, "synergize", explaining the joke.

**Voice example:** "ape in, check back tomorrow. or don't. we're not your mom."

### calm (defi, infra, mature products)

**Words to use:** measured, patient, quiet. Never urgent unless the situation genuinely is. Full sentences. Focus on the user's outcome, not the product's features.

**Words to avoid:** urgency words, countdown timers, "limited time", "act now", anything that manufactures stakes.

**Voice example:** "Your yield has accrued for 30 days. Claim when you're ready."

### electric / weird (memecoin, experimental)

**Words to use:** unexpected, punchy, a little off-kilter. Short sentences. Sudden switches in register. Don't be afraid to break the fourth wall.

**Words to avoid:** explaining yourself. If you feel the need to add "just kidding" after a joke, cut the joke.

**Voice example:** "this button does something. we're not going to tell you what. click it."

## Date format

Use ISO format for the date: `{{DATE}}` → `2026-04-11`. Not "April 11, 2026" — that doesn't sort, and tools that read `brand.md` benefit from parseable dates.

## One tip on section order

The palette goes first, then typography, then gradients, then tone/voice. This ordering isn't arbitrary — it reflects how often each section is referenced. Developers and `frontend-design-guidelines` will hit the palette 100× more than the tone section. Put the highest-traffic info on top.
